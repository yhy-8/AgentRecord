# AgentRecord 设计基线

> 状态：产品与架构基线，确认于 2026-07-16。README 面向使用；本文定义不可被实现细节悄悄改变的产品边界。

## 1. 确定定位

AgentRecord 是一个本地优先的个人记录、整理回顾与领域研究系统：

```text
每天自由记录
  ├─ 每日日记总结
  └─ 每日综合新闻 + 记录驱动信息收集
          ↓
每周 / 每月双板块报告
  ├─ 整理与回顾
  └─ 领域探索与研究
          ↓
用户判断 + 下一轮记录
```

它解决的不是“让 AI 随时回答问题”，而是两个更具体的问题：

1. 将零散记录整理成可追溯的阶段回顾，并观察一个人的观点、理念、理想和行为模式如何延续或变化。
2. 根据记录与综合性新闻雷达选择少量领域问题，利用公开资料研究、分析和推演，拓宽用户能够看到的范围。

AI 不替用户决定如何行动。事实、来源、变化和推演可以由系统呈现；价值判断、现实取舍和行动决定必须留给人。

## 2. 交付物边界

### 2.1 每日层

- 用户日记：标准 Markdown 原始记录流，是唯一事实源。
- 日记总结：写入日记顶部 `<summary>`，用于快速浏览；它不是分析日报。
- 信息简报：包含高价值综合新闻和本周记录驱动信息，供周报/月报选择研究领域；它不是人物结论。

系统不生成分析日报。短周期材料不足以稳定观察思想变化，也容易把报告退化成逐条复述。

### 2.2 周报与月报

每份报告固定包含两个一级板块。

第一板块“整理与回顾”包括：

- 本周期事实、行动、决定、问题与进展；
- 观点、理念、理想和关注领域的形成、延续、变化或冲突；
- 有记录依据的行为模式观察。

行为模式属于基于记录的回顾，不得扩展成心理诊断、人格定性或行为教练建议。

第二板块“领域探索与研究”包括：

- 从记录、每日信息雷达或两者结合得到的一至三个公开研究问题；
- 外部事实、反例、适用边界、相邻概念和不同视角；
- 明确标注不确定性的探索性推断。

第二板块不是新闻链接汇编，也不把新闻热度等同于与用户相关。记录驱动主题必须指出由哪些记录引出；纯新闻主题必须说明其拓宽视野的价值。

### 2.3 质量定义

“优秀报告”不能仅靠增加 Agent 数量保证。当前质量门槛是：

- 两个板块由不同生成 Agent 完成，避免一种提示同时承担回顾和研究；
- 每个板块单独审查事实、时期、身份、来源、因果越界和表达；
- 用户记录使用稳定 `R-*` 来源标记，外部事实就近提供可访问链接；
- 第三方搜索结果由中控分配稳定 `W-*` 证据 ID，模型只选择证据，中控负责把已校验 ID 渲染为 URL；
- 记录、外部事实和 AI 推断明确分开；
- 人物画像只有在本周期记录直接支持时才能新增或更新；
- 已有画像按报告周期结束日截断，未来信息不能泄漏到过去报告。

这些约束可以提高稳定性，但不能保证模型每次都给出同样优秀的判断。报告仍是可失败、可重建的派生结果，用户反馈和来源核查是必要组成部分。

## 3. 原始记录与引用

普通输入在回车提交时只捕获一个本地时间，用它同时确定文件名、文件头和记录时间。断网、模型或数据库失败不得阻止记录追加。

日记标准格式和标准记录流继续兼容旧版本。AI 不能修改原始记录；只有以下两个明确操作可以改日记：

- 用户执行删除最后一条记录；
- 程序原子替换 `<summary>` 区域。

新引用只能选择日记。报告允许一直重新生成且固定路径可能被覆盖，不具备稳定唯一身份，因此不能成为新引用来源。

标准引用格式：

```markdown
**14:32 [引用]:** [日记 | 2026-07-06](<2026-07-06.md>)

这里记录由旧日记继续展开的新想法；可以留空。
```

选择、筛选、相对路径、可选关联记录和删除流程保持不变。历史报告引用保留在旧日记中，不做破坏性迁移；分析上下文只展开位于日记目录中的引用，不读取报告引用。

## 4. 人物画像数据库的意义

SQLite 不再承担通用知识图谱。它只保存四类运行数据：

1. 报告运行及手动、计划、重试触发方式；
2. Agent 中间产物和失败原因；
3. 原始记录来源定位与运行使用关系；
4. 跨周期人物画像及用户反馈。

人物画像类别固定为：

- `viewpoint`：对具体问题的观点或判断；
- `principle`：较稳定的理念、判断原则或方法原则；
- `ideal`：希望长期趋近的状态或价值目标；
- `behavior_pattern`：多次记录可观察到的行为习惯或模式；
- `interest`：持续关注或反复探索的领域。

“思维思想”不作为数据库字段，因为含义过宽。画像条目必须带来源、观察时间和置信度，并通过新版本替代旧版本，不能原地篡改历史判断。用户可以认可、否决或修正条目；反馈影响以后分析，不回写旧报告。

## 5. 数据分层

```text
原始事实层
  Records/YYYY-MM-DD.md

每日研究输入层
  AnalysisReports/Information/YYYY-MM-DD.md

周期交付层
  AnalysisReports/Weekly/*_manual.md | *_auto.md
  AnalysisReports/Monthly/*_manual.md | *_auto.md

派生状态层
  AnalysisReports/.analysis.sqlite3
  AnalysisReports/.automation-state.json

诊断层
  Log/AgentRecord.log
```

手动和自动报告使用不同固定路径。同一来源、同一周期重新生成时可以原子覆盖当前报告，因此 Markdown 报告不被视为稳定身份对象。SQLite 的每次 `analysis_runs` 仍有独立运行 ID，用于审计，不用于日记引用。

## 6. 交互基线

记录模式：

```text
普通文字                   立即追加到今日日记
/v [日期]                  查看日记
/ref [日期]                按日期选择并引用日记
/d                         删除今日最后一条记录
/c                         清空终端显示
/h                         显示帮助
/mode                      切换到报告模式
```

报告模式：

```text
/s [日期]                  生成日记顶部总结
/a weekly [日期]           生成手动周报
/a monthly [日期]          生成手动月报
/retry                     独立后台重试全部失败自动任务
/f                         反馈人物画像条目
/m                         永久切换模型
/status                    查看自动任务状态
/h                         显示帮助
/mode                      返回记录模式
```

报告模式不接收普通记录；记录模式不执行报告命令。手动报告仍在当前窗口的单一工作线程中生成。`/retry` 则启动真正分离的进程，当前窗口退出不影响它。

## 7. 自动化可靠性基线

系统计划任务在登录或重启时运行，并每分钟唤醒一次短进程。短进程用小时水位确保每个新小时只执行一次常规缺漏检测；每分钟触发只是为了覆盖调度延迟、休眠恢复、重启和 5 分钟网络重试，不代表每分钟调用模型。

可靠性规则：

- 常规检测固定核对今日信息简报（到达配置时间后）、昨日日记总结、上周自动周报、上月自动月报四项实际产物。
- 不读取桌面锁定状态。锁屏但系统仍运行时照常执行；锁屏导致的断网按网络错误处理。
- 休眠或关机期间不运行，也不记为失败；恢复后由下一分钟短进程完成该小时尚未发生的检测。
- 使用内核持有的跨进程锁；崩溃、强制结束和重启后由操作系统自动释放，不依赖 PID、文件删除或六小时超时。
- 不保存任务完成游标，完成只由实际文件及日记总结内容证明；报告重试可以复用同输入、同模型下已通过校验和审查的阶段及中控检索证据，旧版进度字段自动清除。
- DNS、连接、超时、HTTP 408 和服务端 5xx 在单次请求内有限重试；整项仍失败时，5 分钟后从头重试。HTTP 429 不做秒级连续请求，直接进入 5 分钟限流等待。连续网络失败继续使用 5 分钟间隔，不能退化成每分钟重跑。
- JSON、输出契约或审查失败由中控在同一 Agent 阶段内有限修订：把未通过稿件和具体错误追加到原请求末尾，最多共调用三次；仍失败或其他非网络失败时整项终止。同一有效输入首次失败后在下一个整点自动重试一次，第二次仍失败则暂停自动请求；输入、模型或搜索配置变化后自动解锁，`/retry` 始终允许用户明确重试。分钟检查在门槛前不得持续消耗模型。
- 小时水位只表示该小时是否做过缺漏扫描，不表示任务完成；若短进程在写入小时水位前被终止，下一分钟会重新扫描并由真实产物去重。
- `/retry` 对命令启动时全部失败的可重试自动任务依次执行，不提供单项选择；生成物仍标记为自动，触发方式记录为 `retry`。
- 自动周报和月报只生成已闭合自然周期；手动报告可以选择进行中周期。

## 8. 明确不做

- 不提供 `@AI` 即时聊天。
- 不要求用户把日记填写成结构化表单。
- 不生成分析日报。
- 不允许新记录引用报告。
- 不建设通用知识图谱、自由讨论式 Agent 群或行为教练。
- 不让 AI 静默修改用户原文、替用户确认价值判断或直接决定行动。
- 不为了兼容旧分析数据库保留迁移或兼容代码；结构不符合时直接拒绝使用，Markdown 记录格式继续兼容。

## 9. 变更完成条件

涉及此定位的改动只有同时满足以下条件才算完成：

- 周报/月报固定输出两个板块，且由不同生成 Agent 负责；
- 日报入口和报告引用选择都不存在；
- 画像类别、报告来源与来源日期可审计；
- 四类产物检测、5 分钟网络重试、有限整点内容重试与输入变化解锁、休眠/重启补检、崩溃遗留锁和全量 `/retry` 有自动化测试；
- README、帮助文本、配置注释、设计文档和实际代码一致；
- 记录写入不依赖模型、网络、SQLite 或自动任务锁。
